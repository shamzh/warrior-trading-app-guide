Part I

Orientation

What the program is for, the hard constraints baked into it, and — most importantly — an honest map of what is built today versus what is deliberately deferred.

1.1 — The Thesis

What This Program Is

A personal desktop assistant for applying Ross Cameron's Warrior day-trading strategy while you trade. It watches the market in real time, finds the kind of volatile small-cap momentum stocks Ross trades, checks each potential trade against his entry rules, and surfaces long-entry recommendations on screen — each with a setup name, an entry, a stop, and a target.

Two principles shape everything it does:

1 · Reduce risk first

Every design choice favors protecting capital over chasing upside, while still generally following Ross's strategy.

2 · Be easy to use

One native window, plain-language recommendations, and a single most-important number always in view.

Hard constraints baked into the program

Constraint · Long-only
It only ever recommends buying. Bear patterns — double tops, head-and-shoulders, breakdowns — are encoded only as anti-signals, never as entries.
Constraint · It proposes, you execute
The program never places orders and never touches your account. It recommends an entry; you decide, and you place the trade in DAS Trader Pro (manually) at your own discretion. A deliberate safety design — you are always the one pulling the trigger.
Constraint · You own the exit and the tape
In v1 the program stops at the entry recommendation. Confirming the entry on Level 2 / Time & Sales and managing the exit (scale out at first profit, move the stop to break-even, bail on a catastrophic stop) are your job, off Ross's rules and your own screen — the program does not do them for you.
Constraint · Small-cap momentum, short holds
$2–$20 stocks, up big on the day, low float, high relative volume — held seconds to a few minutes.
1.2 — Read This First

What's Built vs. What Isn't

The single most important thing to understand: v1 is entry-recommendations-only. The program finds and surfaces the long entries; it does not track your positions or P&L, manage your exits, or read Level 2. And the recommendations come from a deterministic rules engine today — not from Claude reasoning live over the data. Here is the honest map.

✓ BUILT — THE LIVE v1 ENTRY PIPELINE
  • Per-user config layer (your risk anchors + vendor keys)
  • Scanner — Five-Pillars candidate scan on Massive data, true free float via FMP, country via SEC EDGAR
  • ~18 entry-setup detectors (micro pullback, bull flag, ABCD, VWAP reclaim, PM high/pivot breaks, dip trade…)
  • Hard entry gates (four universe pillars + a room-to-run check) + cautionary flags
  • Market-regime classification (informational)
  • The desktop UI, trimmed to the entry-only view (market-conditions, recs feed, scanner board, ticker tape)
  • An offline backtest harness
✗ NOT IN v1 — SHELVED / DEFERRED
  • No position / P&L / account awareness — execution is manual in DAS Trader Pro, which has no usable API. No Record-Fills card, no P&L bar, no positions table.
  • No exit engine — scale-out, break-even / catastrophic stop, MACD-flip / MA-break, bear-pattern exits are doctrine you apply yourself (the code is in-tree, just not wired).
  • No on-screen position sizing — recs show entry / stop / target, no share count.
  • No Level 2 in the program — depth is your own eyes (DAS), not a feed the app consumes.
  • No IBKR — the account was closed; no TWS, no live broker connection.
  • The Claude "brain" — recs are from the rules engine, not live Claude reasoning. Parked.
  • Sub-second print-trigger — not wired; the engine runs in bar-close mode (1-min closes).
  • News pillar — disabled (no vendor). The four price/volume/float pillars carry the scan.
⚠
Why this scope?

The data/execution stack changed materially on 2026-06-02 — IBKR closed, execution moved to manual DAS, and the float vendor moved from Ortex to FMP. Rather than ship half-working position/exit subsystems against a broker the app can't read, v1 was narrowed to the thing it can do well and safely: surface the entries. The shelved subsystems are kept in the codebase for a future version.

Part II

Getting Started

One-time setup and how to launch. Market data comes from Massive; true float from FMP is optional; execution is manual in DAS Trader Pro.

2.1 — One-Time Setup

Prerequisites & Setup

You need a Windows 11 machine, Python 3.12, and the uv package manager. Market data comes from Massive; DAS Trader Pro is your execution platform; an FMP key for true float is optional.

Required

Windows 11, Python 3.12, uv, a Massive (formerly Polygon) Stocks Advanced data subscription, and DAS Trader Pro for execution (with your own Level 2 / Time & Sales subscription).

Optional

An FMP (Financial Modeling Prep) API key for true free float (Pillar 5). Without it, the scanner falls back to the shares-outstanding upper bound and marks ambiguous names needs-confirmation instead of a hard pass.

Install dependencies

# from the project root
uv sync                       # install dependencies into .venv

Create your config

1
Copy config/schema.example.yaml → config/user.yaml. That file is gitignored — your API keys never get committed.
2
Fill in your anchors — the risk values Ross's ratio-based rules use (full position size, daily goal, max daily loss, your "barely-feel-it" per-trade loss, experience tier). See the table below. In v1 these are mostly forward-looking — the subsystems that consume them (sizing, P&L caps) are shelved — but the schema still requires them, so fill them in honestly.
3
Fill in vendor keys: massive.api_key (required for the scanner) and optionally fmp.api_key (true float for Pillar 5).
4
Any ibkr block is ignored (the account is closed) — you can delete it.

The anchor values

AnchorWhat it is
full_position_size_sharesYour full position size (e.g. 1000).
daily_profit_goal_dollarsYour daily goal (e.g. 2000).
max_daily_loss_dollarsYour hard daily stop (e.g. 3000).
personal_barely_feel_loss_dollarsThe per-trade loss you'd "barely feel" — drives risk-based sizing.
user_experience_tierYour experience tier (modulates sizing quality).
🔑
The schema is the source of truth

config/schema.example.yaml is authoritative for exact field names and which are required vs. optional. If a field name in this guide differs from the schema, trust the schema.

↻
Hot reload

Edits to config/user.yaml while the app is running are picked up without a restart. A failed reload keeps the previous config and logs a warning.

2.2 — Boot

Launching the Program

uv run python -m app.main

A native desktop window opens.

⚠
Always launch via uv run

A bare python lacks the timezone data the app needs and will report spurious errors (ZoneInfoNotFound).

A health check you can run without the GUI

uv run python -u scripts/runtime_smoke_test.py   # boot the scan engine headless
⌨
Execution lives in DAS, not in the app

There is no provider to pick and no broker connection to configure. The app surfaces entries; you place and manage every trade manually in DAS Trader Pro. (Earlier versions had Manual / IBKR P&L providers — both are shelved in v1.)

Part III

The Interface

One window, four stacked panels — the cosmetic crawl at the very top, a glanceable read on the market, the recommendations feed, and the scanner board.

3.1 — Top to Bottom

The Screen, Panel by Panel

When the window is open you'll see four panels, stacked top to bottom. The Live Recommendations feed is the heart of v1.

1
Ticker tape (very top) — a decorative news-style crawl of every NASDAQ-listed common stock, alphabetical, with live price and % change. Purely cosmetic; it does not drive any decision.
2
Market Conditions — a glanceable read on the market, not your account (detailed below).
3
Live Recommendations — a card appears whenever a setup fires and clears the gates (detailed below).
4
Scanner board — the autonomous candidate scan. Columns: ticker, country (China/Hong Kong flagged elevated-risk), last price, % change, RVOL, float, verdict. It refreshes itself on a timer — there is no refresh button.
ℹ
No positions table, exits panel, or Record-Fills form

Those belonged to the shelved position/exit subsystems. The P&L meters and loss-cap padlock that used to live in the dashboard were retired with them — the app has no P&L source in v1.

Inside Market Conditions

Regime chip

The market-regime classification (deeply-cold → cold → lukewarm → warm → hot), derived from market breadth. It sets the overall caution tone.

Market-shape flags

Round-trip-prone (leaders giving back gains — favor caution), dispersed attention (too many movers at once), rehab (reserved; not live).

Stand-aside banner

Appears in a deeply-cold regime to suggest favoring standing aside (A-setups only, smallest size).

Recommendation states

Each card shows the setup name, entry / stop / target, the pullback number (if any), a blue-sky tag when the stock is breaking a multi-year high, and any cautions (e.g. "below VWAP", "MACD bearish on 1m"). Its status is one of:

StatusMeaning
PERMITTEDEvery gate passed; this is a tradeable long entry.
BLOCKEDA gate failed; the reason is named (shown in the Suppressed today sidebar).
WARMING_UPA gate can't evaluate yet (indicator data still warming).

A co-firing group (two detectors on the same bar — e.g. micro pullback + half/whole-dollar) renders as a single stacked card with a "Confirmed by:" badge; CH 8-4 calls that combination "the very best version of the setup." (SCALE_IN / SCALE_OUT card styles exist in the UI but don't fire in v1 — they need position awareness, which is out of scope.)

Part IV

Under the Hood

One deterministic pipeline: scan, detect, gate, recommend. In v1 it stops at the recommendation — from there it's you. Here's exactly how each stage decides.

4.1 — The Engine

The Pipeline & Gate Stack

The engine runs one deterministic pipeline. A setup is recognized, the gate stack vets it, and only a recommendation that clears every hard gate is marked PERMITTED. In v1 the pipeline stops at the recommendation — confirming on Level 2, executing in DAS, and managing the exit are yours.

▸ scan the market → build the tradeable universe (Five Pillars)
▸ stream 1-min bars for each candidate
▸ detectors recognize a setup → gate stack vets it
▸ recommendation appears in the feed
▸ you confirm on L2 → execute in DAS → manage the exit yourself

Hard gates — these block a recommendation outright

Hard gate · The four universe pillars (all must be true)
Price $2–$20  ·  % change up ≥ 10% on the day  ·  RVOL ≥ 5× (today's volume vs. its 50-day average)  ·  Float < 20M shares (true free float via FMP when available, else the shares-outstanding upper bound).
Hard gate · Room-to-run
There must be ≥ ~20¢ of room from the entry to the nearest overhead resistance (the next whole dollar above, or the prior high-of-day when it sits above entry). A breakout straight into resistance is suppressed; a genuine blue-sky breakout (new all-time high, no overhead) is treated as unlimited room.
Inert in v1 · Open-position / daily-loss-cap / hijack / rehab
These gates still exist in the composite but are inert in v1 — they need the position and P&L data the app doesn't have. Your broker-side caps in DAS are the real backstop (see §5.2).

Cautionary flags — these inform but never block

You see these on the card and decide for yourself:

MACD on the 1-minute

bearish / warming.

Below VWAP

price below, or VWAP warming.

News

no catalyst / unverified — currently disabled as a pillar.

🔑
Why MACD / VWAP / news are cautions, not gates

Ross's biggest winners repeatedly trade against a cold 1-min MACD, below VWAP, with no news. Price action + supply/demand + Level 2 are primary; these three are advisory. A deliberate 2026-05-29 design decision — see CLAUDE.md.

Context
The daily chart is used as a context filter (overhead resistance, room to run, blue-sky) — never as an entry trigger.

4.3 — Doctrine Reference

Exits — Your Job in v1

The program doesn't manage exits in v1, but the rules it's built around are the ones to apply yourself, off Ross's doctrine and your own screen. (The exit engine code is in-tree, just not wired into the running app.)

Exit · First-profit scale-out + break-even stop
The cleanest hardwired rule in the course: at the first profit target, sell a portion, then move the stop on the runner to break-even.
Exit · Catastrophic stop
Honor your hard stop immediately on the print — these stocks move multiples of the stop width within a single bar, so don't wait for a candle to close.
Exit · Discretionary signals
MACD flip, 9-EMA / VWAP break, 50%-retrace give-back, and the bear-pattern confirmations (double top, head-and-shoulders, top reversal, LOD break) as exit cues, consistent with long-only.
ℹ
In a future version these become live signals

The shelved exit engine emits all of the above as on-screen, urgency-ranked signals (with an alarm on a catastrophic stop). In v1 you run them by hand.

Part V

Trading With It

The full loop of a session — and the layered guardrails that stand between you and a bad day, in priority order.

5.1 — The Loop

A Session, Start to Finish

1
Before the open: in DAS Trader Pro, set your daily max-loss and max position / order size (see §5.2 — do this every day, it's your real backstop). Open your charts and your Level 2 + Time & Sales. Launch the app and glance at Market Conditions to confirm the regime and that your config loaded.
2
Watch the scanner board fill in and the recommendations feed.
3
On a PERMITTED recommendation you like: confirm it on Level 2 / the tape (the app won't do this for you), then place the trade in DAS at (or near) the suggested entry, with the suggested stop, at a size you've chosen.
4
Manage the exit yourself off Ross's rules: take the first-profit scale-out, move the runner's stop to break-even, and act immediately on a catastrophic stop. The program won't prompt these in v1.
5
Mind the regime: in a deeply-cold regime the stand-aside banner is your cue to be more selective (A-setups, smallest size).
6
Stop when you decide. The program runs whenever you run it; there's no forced time window.
5.2 — Guardrails

Risk Controls & Guardrails

In priority order — platform-enforced limits first, because in a "hijacked" emotional state a trader overrides in-app warnings.

Priority 1 · Broker-side caps — your primary defense
Set a daily max-loss and a max position / order size directly in DAS Trader Pro. Non-negotiable. In v1 the app has no in-app loss padlock that actually fires — it can't see your P&L — which makes the DAS-side caps the only enforced guardrail. Set them every day.
Priority 2 · The universe pillars + room-to-run gate
Keep the program from ever surfacing a name that isn't a clean small-cap-momentum long with room above.
Priority 3 · Your own exit discipline
Scale-out + break-even stop and the catastrophic stop, applied by hand (§4.3).
Priority 4 · Regime awareness
Be smaller / more selective in cold or round-trip-prone markets.
⚠
Shelved / deferred guardrails

The in-app daily-loss-cap padlock, the trader-rehab state machine, and a hijack-state gate exist in the codebase but are not enforced in v1 (no live P&L source). Until they are, your DAS-side caps cover that ground.

Tools & Limits

The offline backtest harness, an honest accounting of what is not built, common fixes, and a map of the key files.

6.1 — Looking Back

The Backtest Harness

An offline tool to approximate how the entry engine would have performed on past pre-market sessions. It replays historical 1-minute bars through the same detectors and gates the live app uses, and — to produce a P&L estimate only — simulates a simple exit model that the live v1 app does not run.

# last 5 trading sessions, 7-9am ET, default 0.20% slippage
uv run python -u scripts/backtest_premarket.py --days 5 --out report.md

# a specific past week (walks back from --asof, which is excluded)
uv run python -u scripts/backtest_premarket.py --days 5 --asof 2026-05-22 --out prior.md

# tune the realism / window
uv run python -u scripts/backtest_premarket.py --slippage-bps 35 --window 04:00-09:30

Three fill models — so you see a range, not one misleading number

Optimistic

Perfect fills — the upper bound.

Realistic

Tunable slippage on market-style fills — the central estimate (--slippage-bps, default 20 = 0.20%).

Conservative

Pessimistic fills — the lower bound.

It also reports peak capital deployed per day (the most buying power tied up at once).

⚠
It is an approximation — read it that way

It cannot model Level 2; true sub-second fills are coarsened to 1-minute bars (so fill prices carry real error — that's why there are three models); and the daily universe is reconstructed from each day's movers (so it can miss pre-market-only runners that faded). Read it as "roughly green or red, which setups fired, how often stops vs. targets hit" — not "you would have made exactly $X" — and remember its exit model is a backtest convenience, not how you'll manage trades live.
